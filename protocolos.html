TCP/IP y NetBios

TCP/IP

(Transmission Control Protocol/Internet Protocol) es el conjunto de protocolos que permite la comunicación y transmisión de datos en redes como Internet.

NetBios

(Network Basic Input/Output System) es un protocolo más antiguo diseñado para permitir la comunicación en redes locales (LAN).

Diferencias entre ambos

CP/IP es un protocolo global, diseñado para redes distribuidas (como Internet), mientras que NetBIOS es más adecuado para redes locales.
TCP/IP maneja toda la transmisión de datos, mientras que NetBIOS actúa como una capa adicional para facilitar ciertas funciones de red local, como la identificación de nombres de dispositivos.

Paquete de datos TCP/IP y Flag

Un paquete de datos en TCP/IP tiene dos partes principales


Encabezado (header): Contiene información sobre el origen, destino, tamaño del paquete y otros detalles necesarios para la transmisión.
Datos (payload): Es la información que se está enviando, como un archivo, texto o una solicitud web.
En TCP/IP, el paquete se organiza en capas:
1. Capa de Red (IP):
Encabezado IP: Incluye:
Dirección IP de origen y destino: Define los puntos de inicio y destino de los datos.
Versión IP: Indica si se usa IPv4 o IPv6.
Tamaño del paquete: Indica cuántos bytes incluye.
Tiempo de vida (TTL): Límite de saltos antes de descartar el paquete.
Checksum: Un mecanismo para verificar la integridad del encabezado.
2. Capa de Transporte (TCP o UDP):
Encabezado TCP (si se usa TCP):
Número de secuencia: Indica el orden de los segmentos de datos.
Número de acuse de recibo: Confirma la recepción de datos.
Puerto de origen y destino: Define los puntos de comunicación a nivel de aplicaciones.
Tamaño de la ventana: Controla el flujo de datos.
Flags: Indicadores de control que ayudan a gestionar la conexión.


¿Qué es un flag en TCP/IP?


Un flag es un bit dentro del encabezado TCP que señala ciertas condiciones o controles en una conexión. Los flags son cruciales para administrar el proceso de conexión y la transferencia de datos.

Diferencias entre IPV4 e IPV6

Las principales diferencias entre IPv4 e IPv6 son las siguientes

- IPv4: Utiliza direcciones de 32 bits, lo que permite un total de aproximadamente 4.3 mil millones de direcciones
- IPv6: Usa direcciones de 128 bits, lo que proporciona un total de aproximadamente 340 sextillones de direcciones
- IPv4: Se representa en formato decimal, con cuatro octetos separados por puntos (ej. 192.168.1.1).
- Pv6: Se representa en formato hexadecimal, con ocho grupos de cuatro dígitos separados por dos puntos
- Pv4: Debido al rápido crecimiento de dispositivos conectados, las direcciones IPv4 se están agotando.
- IPv6: Ofrece un número mucho mayor de direcciones, solucionando el problema del agotamiento de IPv4.
- IPv4: La configuración puede ser manual o mediante DHCP (Protocolo de Configuración Dinámica de Host).
- IPv6: Incluye autoconfiguración sin estado (SLAAC), lo que permite a los dispositivos conectarse a una red sin necesidad de un servidor DHCP.
- Pv4: Tiene una estructura de encabezado más simple, pero con menos espacio para opciones avanzadas.
- IPv6: El encabezado es más grande y más complejo, pero está diseñado para ser más eficiente en el enrutamiento y la gestión del tráfico.
- IPv4: La seguridad depende principalmente de protocolos adicionales como IPSec, que es opcional.
- IPv6: Tiene soporte nativo para IPSec, lo que significa que la seguridad está integrada en el protocolo desde el principio.

Estándar IEEE 802.4.

Es un protocolo de red que forma parte de la familia de estándares IEEE 802, desarrollado específicamente para redes de área local (LAN) que utilizan el acceso por Token Bus.
El 802.4 implementa el método de acceso de Token Bus, en el que los dispositivos conectados a la red forman un bus lógico y el control de acceso a la red se pasa de un nodo a otro mediante un "token". Solo el dispositivo que posee el token puede transmitir datos, lo que evita colisiones y garantiza un uso ordenado de la red.
